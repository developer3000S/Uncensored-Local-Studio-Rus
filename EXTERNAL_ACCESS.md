# Настройка внешнего доступа к Web UI

## Обзор

По умолчанию Uncensored AI Studio доступен только через `http://localhost:14200`. 
Эта инструкция поможет настроить доступ из локальной сети и из интернета.

## Доступ из локальной сети (LAN)

Сервер уже настроен слушать на всех сетевых интерфейсах (`0.0.0.0`). 
После запуска в консоли вы увидите сообщение вида:

```
Frontend : http://localhost:14200
Frontend (LAN): http://192.168.1.100:14200 (accessible from local network)
```

其他设备 в вашей локальной сети могут получить доступ к Web UI, используя IP-адрес вашей машины.

### Шаги:
1. Узнайте IP-адрес вашей машины в локальной сети:
   - **Windows**: `ipconfig` (ищите IPv4 Address)
   - **Linux/macOS**: `ip a` или `ifconfig`

2. На других устройствах в той же сети откройте браузер и перейдите по адресу:
   `http://<VAШ_IP_АДРЕС>:14200`

3. Если доступ заблокирован, проверьте настройки файрвола.

## Доступ из интернета (вне локальной сети)

Для доступа из интернета нужно настроить перенаправление портов на роутере.

### Шаги:

1. **Узнайте свой внешний IP-адрес**:
   - Откройте `https://whatismyipaddress.com` или выполните `curl ifconfig.me`

2. **Настройте перенаправление портов на роутере (Port Forwarding)**:
   - Войти в панель управления роутером (обычно `http://192.168.1.1`)
   - Найдите раздел "Port Forwarding" или "Виртуальный сервер"
   - Добавьте правило:
     - Внешний порт: `14200` (или ваш кастомный порт)
     - Внутренний IP: IP-адрес машины с AI Studio
     - Внутренний порт: `14200`
     - Протокол: TCP (или TCP/UDP)
   - Сохраните настройки

3. **Проверьте файрвол на вашей машине**:
   
   **Linux (UFW)**:
   ```bash
   sudo ufw allow 14200/tcp
   ```
   
   **Linux (firewalld)**:
   ```bash
   sudo firewall-cmd --add-port=14200/tcp --permanent
   sudo firewall-cmd --reload
   ```
   
   **Windows**:
   - Откройте "Брандмауэр Windows"
   - Создайте новое правило для входящих подключений
   - Разрешите порт TCP 14200

4. **Проверьте доступ**:
   - С другого устройства (вне вашей сети) откройте: `http://<ВАШ_ВНЕШНИЙ_IP>:14200`

## Настройка через переменные окружения

Вы можете управлять хостом через переменные окружения в файле `.env`:

```bash
# Файл .env в корне проекта
FRONTED_PORT=14200
BACKEND_HOST=0.0.0.0  # Разрешает внешний доступ
API_GPU=8080
TEXT_API=10086
```

### Доступные опции BACKEND_HOST:
- `0.0.0.0` - слушать на всех интерфейсах (доступ из локальной сети и интернета)
- `127.0.0.1` - слушать только локально (только localhost)
- `<конкретный_IP>` - слушать только на указанном IP-адресе

## Дополнительные порты

Если вы хотите, чтобы API бэкендов тоже были доступны извне:

- **Image API**: Порт по умолчанию `8080`
- **Text API**: Порт по умолчанию `10086`

Для доступа к ним из интернета, настройте перенаправление этих портов на роутере.

## Важно!

1. **Безопасность**: Открывая доступ к AI Studio из интернета, убедитесь, что:
   - Вы используете безопасный пароль для вашей сети
   - Ваш роутер имеет надежный пароль администратора
   - Рассмотрите возможность использования VPN вместо прямого доступа

2. **IP-адрес**: Если ваш интернет-провайдер использует динамический IP, ваш внешний адрес может меняться. Рассмотрите использование:
   - Динамического DNS (No-IP, DuckDNS и т.д.)
   - или использования VPN сервиса

3. **Провайдеры**: Некоторые интернет-провайдеры блокируют входящие порты. В этом случае VPN - единственное решение.

## Устранение проблем

### Не работает доступ из локальной сети:
- Проверьте, что сервер запущен и видите сообщение `Frontend (LAN): ...`
- Проверьте, что файрвол не блокирует порт
- Убедитесь, что оба устройства в одной сети

### Не работает доступ из интернета:
- Проверьте настройки Port Forwarding на роутере
- Проверьте внешний IP-адрес (он мог измениться)
- Проверьте файрвол на вашей машине
- Попробуйте отключить роутер и включить снова

### Порт занят:
Если порт 14200 занят, сервер автоматически выберет другой порт в диапазоне 14201-14999.
Вы увидите сообщение: `Порт фронтенда 14200 занят; используется 14201 вместо этого.`
